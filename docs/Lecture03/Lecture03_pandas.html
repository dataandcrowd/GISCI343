Part 1: Selecting and Filtering

Series vs DataFrame: Why the Brackets Matter

Single brackets give you a Series (just the values, like a list with an index). Double brackets give you a DataFrame (a mini table that keeps the column header).

df['name']          # Series
df[['name']]        # DataFrame (one column, still a table)

The easiest way to see the difference is to run both and look at the output. A Series prints as a plain list of values. A DataFrame prints with the column name on top, like a table.

Grabbing Multiple Columns

To grab two or more columns, put the names inside a list, which goes inside the brackets:

df[['name', 'age']]   # DataFrame with two columns

You cannot do this with single brackets because single brackets only accept one column name (a string), not a list. So the pattern is always: double brackets when you want columns back as a table.

A quick mental shortcut: if you are passing a list of column names, you need the outer brackets too, which is why you end up with two layers of brackets.

Selecting Columns

# Single column (returns a Series)
queen_st = ped['45 Queen Street']

# Single column (returns a DataFrame)
queen_st_df = ped[['45 Queen Street']]

# Multiple columns
subset = ped[['Date', 'Time', '45 Queen Street', '150 K Road']]

Single brackets ['col'] returns a Series; double brackets [['col']] returns a DataFrame.

loc and iloc

  • .loc[]: label-based selection
  • .iloc[]: integer position-based selection
# iloc: by position
ped.iloc[0:5, 0:4]  # First 5 rows, first 4 columns

# loc: by label
ped.loc[0:2, ['Date', 'Time', '45 Queen Street']]

Key difference:

  • .iloc[0:3] returns rows 0, 1, 2 (excludes 3)
  • .loc[0:3] returns rows 0, 1, 2, 3 (includes 3)

Boolean Indexing (Filtering)

Uses conditionals from last lecture:

# Hours where Queen Street had > 500 pedestrians
busy = ped[ped['45 Queen Street'] > 500]

# Multiple conditions (AND) - use &
both_busy = ped[
    (ped['45 Queen Street'] > 500) &
    (ped['150 K Road'] > 200)
]

# Multiple conditions (OR) - use |
waterfront = ped[
    (ped['107 Quay Street'] > 300) |
    (ped['Te Ara Tahuhu Walkway'] > 300)
]

The query() Method

Cleaner syntax for filtering (column names with spaces need backticks):

ped.query('`45 Queen Street` > 500')

# Multiple conditions
ped.query('`45 Queen Street` > 500 and `150 K Road` > 200')

# Using external variables with @
threshold = 300
ped.query('`107 Quay Street` > @threshold')

Filtering by Date and Time

# Ensure Date is datetime
ped['Date'] = pd.to_datetime(ped['Date'])

# Filter for March 2024
march = ped[ped['Date'].dt.month == 3]

# Weekdays only (Monday=0 to Friday=4)
weekdays = ped[ped['Date'].dt.dayofweek < 5]

# Morning peak hours
morning_peak = ped[ped['Time'].isin(['7:00-7:59', '8:00-8:59', '9:00-9:59'])]

Part 2: Don’t Forget .copy()

The Problem

Assignment in Python creates a reference, not a copy:

pandas_df_new = pandas_df   # NOT a copy!
pandas_df_new['d'] = [13, 14, 15, 16]

print(pandas_df)  # The original is also modified!
   a  b   d
0  1  5  13
1  2  6  14
2  3  7  15
3  4  8  16

Both variables point to the same object in memory.

The Solution: .copy()

pandas_df_new = pandas_df.copy()  # A true, independent copy
pandas_df_new['d'] = [13, 14, 15, 16]

print(pandas_df)      # Unchanged!
print(pandas_df_new)  # Only this one has column 'd'

Rule of thumb: if you plan to modify a filtered or sliced DataFrame, always call .copy() first.

# Safe pattern for working with subsets
march_data = ped[ped['Date'].dt.month == 3].copy()
march_data['total'] = march_data['45 Queen Street'] + march_data['150 K Road']

Part 3: Manipulating Data

Creating New Columns

df = ped.copy()
df['Date'] = pd.to_datetime(df['Date'])

# Extract temporal features
df['Month'] = df['Date'].dt.month
df['DayOfWeek'] = df['Date'].dt.day_name()
df['IsWeekend'] = df['Date'].dt.dayofweek >= 5

# Extract the starting hour from the Time string
df['Hour'] = df['Time'].str.split(':').str[0].astype(int)

# Total pedestrians across all sensors
location_cols = df.columns[2:22]
df['Total'] = df[location_cols].sum(axis=1)

Conditional Columns

Using .apply() with a custom function:

def time_of_day(hour):
    if hour < 7:
        return 'Early Morning'
    elif hour < 10:
        return 'Morning Peak'
    elif hour < 16:
        return 'Midday'
    elif hour < 19:
        return 'Evening Peak'
    else:
        return 'Night'

df['Period'] = df['Hour'].apply(time_of_day)

Or using np.where() for binary classification:

import numpy as np
df['QueenSt_Busy'] = np.where(df['45 Queen Street'] > 300, 'Busy', 'Quiet')

Part 4: Groupby and Aggregation

The Split-Apply-Combine Pattern

Source: Soner Yildirim, Towards Data Science

  • Split data into groups based on some criterion
  • Apply a function to each group independently
  • Combine the results into a new data structure

Basic Groupby

# Average pedestrian count on Queen Street by day of week
df.groupby('DayOfWeek')['45 Queen Street'].mean()
DayOfWeek
Friday       285.3
Monday       270.1
Saturday     310.5
Sunday       195.2
Thursday     278.9
Tuesday      265.4
Wednesday    272.8

Multiple Aggregations

# Multiple stats for one column
df.groupby('DayOfWeek')['45 Queen Street'].agg(['mean', 'median', 'max', 'count'])

# Different stats for different columns
df.groupby('Period').agg({
    '45 Queen Street': ['mean', 'max'],
    '150 K Road': ['mean', 'max'],
    '107 Quay Street': ['mean', 'max']
}).round(1)

Grouping by Multiple Variables

# Average count by day of week AND time period
cross_tab = df.groupby(['DayOfWeek', 'Period'])['45 Queen Street'].mean().round(1)

# Unstack for readability (rows = days, columns = periods)
cross_tab.unstack()

This is essentially a cross-tabulation: how does foot traffic vary by both day and time?

Part 5: Reshaping Data

Wide vs Long: Why Does It Matter?

Wide format is what most people are used to: each variable gets its own column. Think of a spreadsheet where columns are 2021_pop, 2022_pop, 2023_pop. Easy to read as a human.

Long format stacks those into rows instead, so you get a year column and a pop column. Fewer columns, more rows.

Why does this matter? Because plotting libraries and many pandas operations prefer long format. If you want to plot population over time and colour by region, tools like seaborn expect one column for the value and one column for the grouping variable. Wide format makes that awkward.

The short version: melt for analysis, pivot for presentation.

Melt and Pivot

  • melt(): wide to long format
  • pivot() / pivot_table(): long to wide format

Melt: Wide to Long

The pedestrian data has 20 location columns. We can “melt” them into two columns: Location and Count.

ped_long = df.melt(
    id_vars=['Date', 'Time', 'Hour', 'Month', 'DayOfWeek', 'IsWeekend', 'Period'],
    value_vars=location_cols,
    var_name='Location',
    value_name='Count'
)

print(f"Wide shape: {df.shape}")
print(f"Long shape: {ped_long.shape}")
Wide shape: (8796, 30)
Long shape: (175920, 9)

8,796 rows x 20 locations = 175,920 rows

Pivot Table: Long to Wide

# Average hourly count by location and time period
pivot = ped_long.pivot_table(
    values='Count',
    index='Location',
    columns='Period',
    aggfunc='mean'
).round(1)

print(pivot)

Rule of thumb: making a table? Pivot. Making a plot? Melt.

Part 6: Handling Missing Data

Why Missing Data Matters

Real-world sensor data is messy. Sensors malfunction, get obscured, or lose power. If you ignore missing values or handle them carelessly, your results can be silently wrong.

Consider the Auckland pedestrian sensors:

  • A sensor that fails during heavy rain will undercount pedestrians precisely when behaviour changes most. Filling those gaps with the overall mean pretends rain makes no difference.
  • A sensor not yet installed at the start of the year has no observations. Filling with zero implies “we saw nobody,” which is not the same as “we were not looking.”
  • A sensor that briefly loses power one afternoon is genuinely missing at random. Interpolation or a mean fill is reasonable here.

The method you choose should match why the data is missing.

Three Types of Missingness

Type Example Safe to fill?
Missing at random Sensor glitch on a Tuesday afternoon Yes, mean or interpolation
Missing systematically Sensor fails every time it rains Filling naively biases results
Missing by design Sensor installed in March, no Jan/Feb data Keep as NaN, do not fill with zero

In geospatial work specifically, missing coordinates usually mean you must drop the record, because there is no meaningful way to impute a location. Missing attributes, by contrast, can sometimes be filled using spatial neighbours.

Whatever you do, document it in your code or report.

Detecting Missing Values

# Count missing values per column
ped.isnull().sum()

# Total missing
ped.isnull().sum().sum()

# Percentage missing per column
(ped.isnull().sum() / len(ped) * 100).round(2)

Filling Missing Values

Different approaches for different situations:

# Fill with zero (sensor was working, no one walked by)
ped_zero = ped.fillna(0)

# Fill with column mean (sensor was temporarily offline)
ped_mean = ped.copy()
ped_mean['45 Queen Street'] = ped_mean['45 Queen Street'].fillna(
    ped_mean['45 Queen Street'].mean()
)

# Forward fill (use previous hour's value for brief gaps)
ped_ffill = ped.fillna(method='ffill')

# Interpolation (best for time series)
ped_interp = ped.copy()
ped_interp['45 Queen Street'] = ped_interp['45 Queen Street'].interpolate(
    method='linear'
)

Think Before You Fill!

  • Missing at random (sensor malfunction): safe to fill with mean/interpolation
  • Missing systematically (fails during rain): filling naively biases your results
  • Missing by design (sensor not yet installed): keep as NaN, do not fill with zero

Always document what you did and why.

Part 7: Merging DataFrames

Join Types

Example: Two DataFrames

df1 = pd.DataFrame({
    'ID': [1, 2, 3, 4],
    'Name': ['Tom', 'Nick', 'Krish', 'Jack'],
    'Age': [20, 21, 19, 18]
})

df2 = pd.DataFrame({
    'ID': [3, 4, 5, 6],
    'Subject': ['Maths', 'English', 'Science', 'History'],
    'Score': [88, 75, 90, 85]
})

Merging

# Inner: only matching rows
pd.merge(df1, df2, on='ID', how='inner')

# Left: keep all from left
pd.merge(df1, df2, on='ID', how='left')

# Outer: keep everything
pd.merge(df1, df2, on='ID', how='outer')

Merging with the Pedestrian Data

Once we melt to long format, we can merge with sensor metadata:

sensor_metadata = pd.DataFrame({
    'Location': ['107 Quay Street', '45 Queen Street', '150 K Road'],
    'Area': ['Waterfront', 'CBD', 'K Road'],
    'Latitude': [-36.8430, -36.8442, -36.8580],
    'Longitude': [174.7685, 174.7659, 174.7560]
})

merged = pd.merge(ped_long, sensor_metadata, on='Location', how='inner')

This adds spatial attributes (coordinates, area type) to each pedestrian count record. In Section 2.4, you will do this with geometry using spatial joins in GeoPandas.

Part 8: Quick Visualisation

What is matplotlib?

matplotlib is a Python library for making charts and graphs. Think of it as Python’s drawing toolkit for data.

import matplotlib.pyplot as plt   # We always import it like this

The basic recipe for any plot has three steps:

  1. Create a figure (the canvas)
  2. Draw something on it (a line, bars, dots, etc.)
  3. Show it with plt.show()

Step-by-step: Your First Plot

import matplotlib.pyplot as plt

# Step 1: Create a figure and set its size (width, height in inches)
plt.figure(figsize=(8, 4))

# Step 2: Draw a simple line plot
x = [1, 2, 3, 4, 5]
y = [10, 40, 25, 50, 35]
plt.plot(x, y)

# Step 3: Show it
plt.show()

That is it! You have made a plot. Now let us make it look better.

Adding Labels and Titles

Labels tell your reader what they are looking at. Always label your axes.

plt.figure(figsize=(8, 4))
plt.plot(x, y, marker='o', color='steelblue')   # marker='o' adds dots at each point

plt.xlabel('Day')              # Label for the x-axis
plt.ylabel('Pedestrian Count') # Label for the y-axis
plt.title('Daily Counts')      # Title at the top
plt.grid(True, alpha=0.3)      # Light gridlines (alpha = transparency)
plt.tight_layout()             # Prevent labels from being cut off
plt.show()

Common Plot Types

What you want Code kind= for pandas
Line chart plt.plot(x, y) 'line'
Bar chart (vertical) plt.bar(x, y) 'bar'
Bar chart (horizontal) plt.barh(x, y) 'barh'
Scatter plot plt.scatter(x, y) 'scatter'
Histogram plt.hist(values, bins=20) 'hist'

pandas DataFrames can call these directly, e.g. df['col'].plot(kind='hist').
